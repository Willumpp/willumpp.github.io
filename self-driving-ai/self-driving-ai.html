Overview

A coursework project for my A-Level computer science unit. The project looks at testing "vehicles" in a simulation with different algorithms to navigate them accross a track. The algorithms used are reinforcement learning, A* pathfinfing, and a heuristic based algorithm. All results are stored in an external SQLite database for the user to analyse as they see fit. Testing can take place on one of the inbuilt tracks or a custom made track with the track building tool.

Algorithms

This project makes use of three different algorithms: A machine learning/reinforcement learning/genetic algorithm, an A* pathfinding algorithm, and a wandering algorithm.

Genetic Algorithm

This algorithm is based on the idea of evolution and trial and error within diverse generations of vehicles. For instance, from a set of 30 vehicles with completely random parameters for determing when to turn at what wall distance, at least one is going to get a bit far. If that succeeds, this vehicle is selected for the next generation. And this cycle continues until a working car makes it to the end.

A* Pathfinding Algorithm

Upon starting the simulation a path is found through the track using an A* pathfinding algorihtm. See Pathfinder for another project using this. The path generated from the algorithm is then interpolated between nodes with multiple cubic b√©zier curves in order to form a continuous path.

The vehicle offroading is accounted for in the results database. The longer it spends offroading, the more its recognised as an error in driving.

Credit to this source to help me with the steering behaviours.

Wandering/Obstacle Avoidance

From the vehicle, three "rays" are casted out to determine the distance of the closest walls in each direction. Using this, the vehicle can steer away form walls, speed up, or slow down. The result is a very careful and error free driver for the track.

GUI

The GUI in use is entirely made from scratch with pygame. This is the library I put together whilst making this project.

The library offers a range of widgets to place on the screen. These include text input fields, buttons, and pictures. Addiitonally, it can update any objects on the screen every frame. Each scene is split into a "layer" where only active layers are exectued every frame.

Settings

Theres a variety of settings to choose from and change for when testing these vehicles. Settings range from cosmetics to changing the physics of the test environment; so, you can test a variety of situations.

(Elasticity and friction have no functionality).

Additionally each algorithm has their own unique setttings. For example, this is the genetic algorithm settings.

Mutation rate refers to the chance that the best performing vehicle gets chosen for mutation. The mutation chance refers to the proportion of genes that are inherited for each vehicle in the next generation.

Databases

Upon running Main.py for the first time you are met with a login menu. Here you either sign or or create a new account. This was mainly done as it was a requirement for the coursework, but it also has a helpful feature. Having accounts means tracks and test results can be unique to the user. The SQLite database is stored in a seperate location for the user so they know it is their data.
